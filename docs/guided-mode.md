# Guided Mode v0.6

Guided Mode turns Flow Covenant Runtime into a step-by-step walkthrough for people who do not want to learn the entire model before using it.

## Purpose

Guided Mode helps a user, team, facilitator, or community group move through the full FCR sequence without losing the claim-safe boundary, consent checks, repair paths, or receipt discipline.

## Guided sequence

```text
Context -> Signal -> Invitation -> Affected Parties -> Consent -> Gates -> Alignment -> Repair -> Flow -> Receipt -> Renewal -> Evidence -> Finish
```

## What it does

- Keeps the user focused on one question at a time.
- Writes directly into the same receipt and repair-plan structures used by the rest of the app.
- Keeps governance gates visible before action settles.
- Shows risk, gate counts, reversibility label, and completion progress.
- Saves guided sessions locally using the same scenario library.

## Claim-safe boundary

Guided Mode does not automate moral, legal, medical, scientific, or spiritual authority. It is a structured reflection and systems-design workflow.

## Facilitation stance

The facilitator should preserve calm, consent, clarity, evidence, and repair. Watch/block gates are not failures; they are signals that a better repair path or smaller reversible action is needed.

## Acceptance checklist

A guided session is ready to settle only when it has:

- A named scenario and signal.
- A clear invitation and refusal path.
- Affected parties named.
- Consent or authority clarified.
- All six gates reviewed.
- Repair and reversibility path described.
- Evidence, assumptions, and unknowns separated.
- A final receipt saved, copied, or exported.
