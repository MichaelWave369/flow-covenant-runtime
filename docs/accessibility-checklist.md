# Accessibility Checklist

Use this checklist before calling a Flow Covenant artifact public-ready.

## Interface

- All interactive controls are reachable by keyboard.
- Buttons and links have clear text labels.
- Form fields have visible labels.
- Color is not the only way status is communicated.
- Important status changes are expressed in text.
- Long pages keep headings and section order clear.

## Language

- Claim boundaries are visible and plain-language.
- The app avoids shame-based language.
- Repair language does not minimize harm.
- Evidence, assumptions, and unknowns are clearly separated.
- The model is not framed as medical, legal, scientific, or spiritual authority.

## Workshop use

- Participants can pause or opt out.
- A facilitator names the claim-safe boundary at the start.
- People affected by a scenario are not spoken over.
- Watch/block gate items are not rushed into settlement.
- A written receipt captures the decision, repair path, and review date.

## Local-first boundary

- No accounts are required.
- No backend is required.
- No hidden network calls are introduced.
- Library data stays in the browser unless the user exports it.
