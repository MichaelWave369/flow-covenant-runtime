## Claim-safe boundary

This is a philosophy and systems-design model, not a claim that physical laws literally want, think, or feel.

FCR does not provide medical advice, legal advice, scientific proof, spiritual authority, or prediction machinery. It is a practical decision and governance scaffold for consent, evidence, repair, reversibility, and receipts.

## Local-first boundary

The app does not require accounts, backend services, hidden network calls, telemetry, or analytics. Saved scenarios and receipts remain in browser localStorage unless the user chooses to export them.

## Stable maintenance rule

Future changes should preserve:

- claim-safe language,
- local-first operation,
- no hidden network calls,
- no private user data collection,
- explicit consent/evidence/repair gates,
- receipt export and replay integrity.

## Next candidate direction

The likely next line is v1.1, focused on import validation previews, printable facilitator packets, additional example receipts, and a documented accessibility pass.
