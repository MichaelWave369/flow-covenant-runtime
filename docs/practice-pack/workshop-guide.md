# Flow Covenant Runtime Workshop Guide

A 60-minute facilitation pattern for using Flow Covenant Runtime in a group setting.

## Purpose

Help a group move a charged decision, unclear promise, risky action, or repair need through a transparent flow: Signal -> Invitation -> Consent -> Alignment -> Flow -> Receipt -> Renewal.

## Facilitator stance

- Keep the claim-safe boundary visible.
- Do not use flow language to avoid accountability.
- Slow the room down when people jump to blame, punishment, or vague optimism.
- Separate facts, assumptions, unknowns, and disputed items.
- Prefer reversible moves when uncertainty is high.

## Materials

- Practice worksheet
- Decision-card deck
- Timer
- Shared notes or whiteboard
- One designated receipt writer

## 60-minute flow

| Time | Segment | Prompt |
| --- | --- | --- |
| 0-5 min | Boundary and purpose | What are we trying to preserve: consent, truth, repair, and future play? |
| 5-15 min | Name the signal | What is actually arising before interpretation or blame? |
| 15-25 min | Map invitation and affected parties | What is being offered, and who is affected? |
| 25-35 min | Run governance gates | Which gates pass, need watch, or block action? |
| 35-45 min | Build repair and reversibility | What can be paused, drafted, tested, rolled back, or repaired? |
| 45-55 min | Write the receipt | What changed, why, what evidence supports it, and what remains unknown? |
| 55-60 min | Renewal | What follow-up proves the field is healthier? |

## Red flags

Pause the session if:

- Harm becomes harder to name.
- Power hides behind soft language.
- Consent is bypassed.
- Evidence is replaced with vibes.
- No useful receipt can be produced.

## Closing receipt

End with one paragraph:

> We noticed ____. We invited ____. Affected parties were ____. The gates showed ____. We chose ____. We will repair or review by ____. Remaining unknowns are ____.
